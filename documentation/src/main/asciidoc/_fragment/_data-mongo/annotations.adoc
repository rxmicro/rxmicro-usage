=== `RxMicro Annotations`

The RxMicro framework supports the following `RxMicro annotations`:

.Supported `RxMicro Annotations`.
[cols="1,3"]
|===
|*Аннотация*|*Описание*

|{RX-MICRO-SRC-ROOT-BLOB-URL}rxmicro-data/src/main/java/io/rxmicro/data/Column.java[`@Column`^]
|Устанавливает mapping между именем поля в `Mongo DB` документе и именем поля класса Java модели.

_(По-умолчанию the RxMicro framework использует имя поля класса Java модели в качестве имени поля в `Mongo DB` документе.
Если имя по каким-либо причина должно отличаться, (например в качестве имени поля `Mongo DB` документа используется ключевое слово Java), то необходимо его указать используя данную аннотацию!)_

|{RX-MICRO-SRC-ROOT-BLOB-URL}rxmicro-data/src/main/java/io/rxmicro/data/ColumnMappingStrategy.java[`@ColumnMappingStrategy`^]
|Устанавливает стратегию формирования имен полей в `Mongo DB` документе, на основании анализа имен полей класса Java модели.

_(Если данной аннотацией аннотирован класс Java модели, то будет использоваться установленная стратегия для *всех* полей в данном классе.
Например, если будет установлена стратегия по-умолчанию: {RX-MICRO-SRC-ROOT-BLOB-URL}rxmicro-model/src/main/java/io/rxmicro/model/MappingStrategy.java#L34[`LOWERCASE_WITH_UNDERSCORED`^],
то полю `parentId` в Java классе будет соответствовать поле `parent_id` в `Mongo DB` документе.)_

|{RX-MICRO-SRC-ROOT-BLOB-URL}rxmicro-data/src/main/java/io/rxmicro/data/DataRepositoryGeneratorConfig.java[`@DataRepositoryGeneratorConfig`^]
|Позволяет конфигурировать процесс генерации репозиториев.

|{RX-MICRO-SRC-ROOT-BLOB-URL}rxmicro-data/src/main/java/io/rxmicro/data/RepeatParameter.java[`@RepeatParameter`^]
|Позволяет установить <<data-mongo-repeat-parameter-section, mapping между одним параметром метода, помеченным данной аннотацией и нескольким универсальными placeholders, которые используются в запросе к `Mongo DB`>>.
