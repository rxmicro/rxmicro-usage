Однако если на базе такого теста построить отчет по степени покрытия тестами исходного кода проекта, данный отчет покажет слабую степень покрытия:

.Отчет покрытия тестами при использовании альтернативы-мока для Mongo репозитория.
image::testing/microservice/alternative/mongo-repo/jacoco-business-logic-only.jpg[]

Такой результат связан с тем, что создав альтернативу-мок для компонента `DataRepository`, в процессе тестирования не используются сгенерированные фреймворком RxMicro классы для работы mongo репозитория.

Если такой результат не является допустимым, то необходимо:

. создавать альтернативу-мок RxMicro компоненту `MongoDatabase`;
. использовать статические методы класса
{RX-MICRO-SRC-ROOT-BLOB-URL}rxmicro-test-mockito/src/main/java/io/rxmicro/test/mockito/mongo/MongoMockFactory.java[`MongoMockFactory`^] для программирования поведения мока.
